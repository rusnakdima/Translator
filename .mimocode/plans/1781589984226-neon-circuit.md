# Plan: Force software rendering for Tauri on WSL2

## Problem
WSL2 has no GPU passthrough by default. Tauri v2's webview (WebKit) loads `libEGL`/`ZINK` which fail because there's no GPU device, causing the app to crash or hang.

## Solution
Create `src-tauri/.env` with environment variables that force software rendering:

```
LIBGL_ALWAYS_SOFTWARE=1
MESA_LOADER_DRIVER_OVERRIDE=swrast
WEBKIT_DISABLE_COMPOSITING_MODE=1
WEBKIT_DISABLE_GPU=1
```

Cargo automatically loads `.env` files in the manifest directory before `cargo run`, so these will be set when the Tauri dev server runs `cargo run --no-default-features`.

## File to create
- `src-tauri/.env`

## Verification
Run `bun tauri dev` — the libEGL/ZINK errors should be gone and the window should appear.
