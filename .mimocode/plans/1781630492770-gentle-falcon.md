## Steps

### 1. Stash unstaged changes (preserve work-in-progress)
```bash
git stash push -m "WIP: local dev changes before merge"
```

### 2. Pull remote (merge remote into local)
```bash
git pull origin main
```
This creates a merge commit combining both histories.

### 3. Inspect for conflicts
If conflicts exist, resolve by keeping **newest version** (local if newer, otherwise remote).

### 4. Restore stashed changes
```bash
git stash pop
```
If conflicts arise from stash, resolve keeping newest.

### 5. Stage and commit merge result
```bash
git add -A
git commit -m "merge: reconcile local and remote changes"
```

### 6. Push merged result
```bash
git push origin main
```

## Conflict Resolution Rule
For any file conflict, keep the **newest version** based on:
- File modification timestamp, or
- Content newer (e.g., higher version numbers)
